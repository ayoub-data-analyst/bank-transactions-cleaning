## Context

FinanceCore SA exports its transactional data daily into a raw CSV file generated by the central banking system. This file contains numerous quality issues (duplicates, mixed formats, incorrect types, outliers) that make it unusable as-is.

This project implements a  **complete 5-step cleaning and enrichment pipeline** , whose output `financecore_clean.csv` will serve as the foundation for the PostgreSQL relational model and the Streamlit dashboard in the following weeks.

| # | Issue                                                | Affected Columns                                      |
| - | ---------------------------------------------------- | ----------------------------------------------------- |
| 1 | Missing values                                       | `segment_client`,`agence`,`score_credit_client` |
| 2 | Duplicates (same `transaction_id`)                 | `transaction_id`                                    |
| 3 | Mixed date formats (`DD/MM/YYYY`vs `YYYY-MM-DD`) | `date_transaction`                                  |
| 4 | Comma decimal separator instead of period            | `montant`                                           |
| 5 | Text suffix `" EUR"`blocking numeric typing        | `solde_avant`                                       |
| 6 | Inconsistent casing                                  | `devise`,`segment_client`                         |
| 7 | Leading/trailing whitespace                          | `agence`                                            |
| 8 | Outliers (extreme amounts, out-of-range scores)      | `montant`,`score_credit_client`                   |
| 9 | Irrelevant column                                    | `taux_interet`                                      |

```

        │
```

## Detailed Steps

### Step 1 — Import & Exploration

* Load the CSV with `pandas`
* Inspect: `shape`, `dtypes`, `head()`, `describe()`
* Calculate and visualize missing value rates per column (bar chart)
* Identify duplicates on `transaction_id`

### Step 2 — Cleaning

| Action             | Column                        | Method                                         |
| ------------------ | ----------------------------- | ---------------------------------------------- |
| Drop unused column | `taux_interet`              | `drop()`                                     |
| Deduplication      | `transaction_id`            | `drop_duplicates(keep="first")`              |
| Date unification   | `date_transaction`          | `pd.to_datetime(errors="coerce")`+`dropna` |
| Decimal fix        | `montant`                   | `str.replace(",", ".")`+`astype(float)`    |
| Strip suffix       | `solde_avant`               | `str.replace("EUR", "")`+`astype(float)`   |
| Case normalization | `devise`                    | `str.upper()`                                |
| Case normalization | `segment_client`            | `str.lower().str.capitalize()`               |
| Imputation         | `segment_client`,`agence` | Mode                                           |
| Strip whitespace   | `agence`                    | `str.strip()`                                |

### Step 3 — Outliers

| Column                  | Method                   | Decision                           |
| ----------------------- | ------------------------ | ---------------------------------- |
| `score_credit_client` | Business rule [300, 850] | Nullify + median imputation        |
| `montant`             | IQR × 1.5               | Keep + flag `is_anomaly_montant` |

### Step 4 — Feature Engineering

| New Column                                       | Description                                                 |
| ------------------------------------------------ | ----------------------------------------------------------- |
| `anne`,`mois`,`trimestre`,`jour_semaine` | Temporal decomposition of `date_transaction`              |
| `montant_eur_verifie`                          | `montant / taux_change_eur`                               |
| `ecart_eur`                                    | `montant_eur − montant_eur_verifie`                      |
| `is_conversion_error`                          | `True`if `\|ecart_eur\| > €1`                            |
| `categorie_risque`                             | `Low`/`Medium`/`High`based on `score_credit_client` |
| `solde_net`                                    | Total credits − total debits per `client_id`             |
| `nb_transactions`                              | Number of transactions per `client_id`                    |
| `montant_moyen`                                | Average amount per `client_id`                            |
| `nb_produits_distincts`                        | Unique products per `client_id`                           |
| `taux_rejet`                                   | % rejected transactions per `agence`                      |

### Step 5 — Export

* Verification: zero missing values on critical columns, correct numeric types
* Export: `financecore_clean.csv` (UTF-8 CSV, no index)
